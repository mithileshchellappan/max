## Security

Please do not report security vulnerabilities through public GitHub issues.

Use GitHub private vulnerability reporting for this repository if it is enabled. If it is not enabled, contact the repository owner privately before sharing proof-of-concept details.

When reporting a vulnerability, include:

- Issue type and affected feature.
- Affected file paths, branch, tag, or commit.
- Required configuration or environment details.
- Reproduction steps.
- Proof-of-concept details when available.
- Expected impact.
